# 6TiSCH Hareketlilik Senaryolarına Katkı Sağlama

Bu projeye katkıda bulunmayı düşündüğünüz için teşekkür ederiz! Zamanınıza ve emeğinize değer veriyoruz.

## Nasıl Katkıda Bulunabilirsiniz?

1.  **Repository'yi Fork Edin**: Bu repository'yi kendi GitHub hesabınıza "fork" ederek başlayın.
2.  **Klonlayın**: Fork ettiğiniz projeyi yerel bilgisayarınıza klonlayın.
3.  **Bir Branch Oluşturun**: Yeni özelliğiniz veya hata düzeltmeniz için yeni bir dal (branch) oluşturun.
    ```bash
    git checkout -b feature/harika-yeni-ozellik
    ```
4.  **Değişiklikleri Yapın**: Kodunuzu geliştirin. Kodun temiz ve iyi belgelenmiş olduğundan emin olun.
5.  **Commit Atın**: Değişikliklerinizi net ve açıklayıcı mesajlarla kaydedin.
    ```bash
    git commit -m "Değişikliklerin açıklaması"
    ```
6.  **Branch'inizi Push Edin**: Değişikliklerinizi kendi fork'unuza gönderin.
    ```bash
    git push origin feature/harika-yeni-ozellik
    ```
7.  **Pull Request (PR) Açın**: Fork'unuzdan ana repository'ye bir Pull Request açın. Yaptığınız değişiklikleri detaylıca açıklayın.

## Kodlama Standartları

*   **Scriptler**: `.js` analiz dosyalarında net değişken isimleri ve yorum satırları kullanın.
*   **Dokümantasyon**: Yeni senaryolar veya özellikler eklerseniz README dosyasını güncelleyin.
*   **İz Dosyaları (Traces)**: Yeni hareket izleri ekliyorsanız, bunları oluşturmak için kullanılan BonnMotion parametrelerini belirtin.

## Sorularınız mı Var?

Herhangi bir sorunuz veya öneriniz varsa, lütfen bir "Issue" açmaktan çekinmeyin!
